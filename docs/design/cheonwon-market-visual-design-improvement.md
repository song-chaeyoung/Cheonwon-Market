# 천원마켓 디자인 개선 방안

## 목적

천원마켓의 시각 디자인을 "친구 모임에서 쓰는 작고 가벼운 예약 장터"로 정리합니다. 이 문서는 UX 플로우가 아니라 로고, 색, 카드, 배지, 버튼, 화면 분위기 같은 시각 기준을 고정하기 위한 문서입니다.

## 현재 근거

- 로고 시안은 `docs/design/logo-favicon-concepts.html`에 있으며, 추천안은 "가격표 태그 + 원화 기호"입니다. 해당 시안은 `--mint: #65c6a4`, `--mint-dark: #278866`를 이미 사용합니다.
- 현재 전역 색상은 `src/app/globals.css`에서 대부분 무채색 OKLCH 값으로 잡혀 있습니다. 깨끗하지만 브랜드 인상이 약합니다.
- 상품 카드는 `src/components/products/product-card.tsx`에서 정사각형 이미지, 둥근 카드, 가격, 상태 배지 구조를 사용합니다.
- 상태 배지는 `src/components/products/status-badge.tsx`에서 판매중 emerald, 예약중 amber, 거래완료 zinc 계열을 사용합니다.
- 입장 화면은 `src/components/access/access-code-screen.tsx`에서 천원마켓 이름과 친구 모임 입장 코드 문구를 이미 가지고 있습니다.
- 비주얼 레퍼런스 보드는 `.lazyweb/quick-references/cheonwon-market-visual-design-2026-06-10/report.html`에 있으며, Karrot, OfferUp, Vinted 이미지를 비교했습니다.

## 디자인 방향 결정

메인 컬러는 초록 계열로 갑니다. 단, 강한 녹색이 아니라 로고 시안의 민트 그린을 메인으로 사용합니다.

```txt
Main:       #65C6A4
Main dark:  #278866
Paper:      #FFFDF8
Soft bg:    #F7FBF8
Text:       #1F2933
Muted:      #687482
Line:       #DFE5EA
Warning:    #F6C453
Danger:     #ED6A5A
```

근거: 로고 시안이 이미 민트 가격표 태그를 중심으로 만들어져 있고, 현재 판매중 상태도 emerald 계열이라 주색과 상태 의미가 충돌하지 않습니다. 또한 Vinted처럼 상품 사진이 중심이 되는 카탈로그형 화면에서는 오렌지보다 민트가 덜 공격적입니다.

## 브랜드 인상

목표 인상은 "따뜻한 민트 라벨 + 깨끗한 중고 카탈로그"입니다.

- 너무 커머스처럼 보이지 않습니다.
- 너무 관리자 도구처럼 건조하지 않습니다.
- 친구 모임 전용이라는 작고 안전한 느낌을 유지합니다.
- 상품 사진이 주인공이고, 브랜드 컬러는 보조 신호로만 반복합니다.

## 로고 적용

### 헤더 로고

헤더에서는 태그 아이콘과 워드마크를 함께 씁니다.

```txt
[민트 가격표 ₩] 천원마켓
               공짜부터 천원까지, 가볍게 예약
```

근거: 첨부 이미지와 `logo-favicon-concepts.html`의 A안은 48px, 32px 크기에서 형태가 잘 유지됩니다.

### 파비콘

파비콘은 태그 형태를 그대로 쓰기보다 단순화합니다.

```txt
민트 바탕 + 흰색 ₩
```

근거: 16px 크기에서는 태그 꼬리와 내부 구멍이 무너질 수 있습니다. 첨부 이미지에서도 alt 형태가 더 안정적으로 보입니다.

## 화면별 개선 방향

### 1. 전역 배경과 표면

현재 `bg-muted/30` 중심의 회색 배경을 따뜻한 오프화이트/민트 기운 배경으로 바꿉니다.

```txt
page background: #F7FBF8
card background: #FFFDF8 또는 #FFFFFF
border: #DFE5EA
```

근거: 현재 `globals.css`의 루트 색상은 대부분 무채색입니다. 로고의 민트가 화면에 자연스럽게 연결되려면 배경에도 아주 약한 따뜻함이 필요합니다.

### 2. 상품 카드

상품 카드는 사진 중심으로 유지하되, 정보 위계를 더 선명하게 만듭니다.

- 이미지는 카드의 가장 큰 면적을 차지합니다.
- 가격은 제목보다 조금 더 강하게 보입니다.
- 상태 배지는 카드 상단 또는 이미지 위에 작게 얹습니다.
- 카드 radius는 8px 전후로 유지합니다.
- shadow는 거의 쓰지 않고 border 중심으로 갑니다.

근거: `product-card.tsx`는 이미 `aspect-square` 이미지 구조를 가지고 있어 사진 중심 카드로 확장하기 쉽습니다. Vinted 레퍼런스도 사진과 일정한 카드 밀도가 핵심입니다.

### 3. 버튼

주요 CTA는 민트 계열 solid 버튼으로 통일합니다.

```txt
primary button: #278866 background + white text
primary hover: 더 어두운 mint dark
secondary: white background + mint border/text
```

근거: 현재 shadcn 기본 primary는 검정 계열이라 로고와 연결되지 않습니다. 메인 CTA가 민트 계열이면 로고와 버튼이 한 브랜드로 보입니다.

### 4. 상태 배지

기존 의미는 유지하되 색을 브랜드 팔레트에 맞춥니다.

```txt
판매중:   mint green
예약중:   soft amber
거래완료: warm gray
```

근거: 현재 `status-badge.tsx`도 판매중/예약중/완료를 emerald/amber/zinc로 구분합니다. 색 의미는 그대로 두고 채도와 배경만 조정하면 기존 정보 구조를 유지할 수 있습니다.

### 5. 입장 화면

입장 화면은 보안 폼보다 초대받은 작은 장터 입구처럼 보이게 합니다.

- 로고 아이콘을 상단에 둡니다.
- 배경은 `#F7FBF8` 또는 아주 연한 민트 톤을 씁니다.
- 폼 카드는 흰색 표면과 얇은 border로 정리합니다.
- 버튼은 민트 solid로 갑니다.

근거: `access-code-screen.tsx`는 이미 "천원마켓", "친구 모임에서 공유한 공통 입장 코드" 문구를 가지고 있어, 시각 톤만 바꿔도 서비스 정체성이 강화됩니다.

### 6. 등록/수정 폼

폼은 흰 표면 위에 민트 포인트를 적게 씁니다.

- 이미지 업로드 영역의 dashed border를 mint 계열로 조정합니다.
- 선택된 이미지/대표 이미지는 민트 라벨을 사용합니다.
- 긴 입력 영역은 회색보다 따뜻한 border를 사용합니다.
- 위험 액션은 tomato `#ED6A5A`로 분리합니다.

근거: Karrot 등록 화면 레퍼런스는 흰 바탕, 굵은 제목, 넓은 입력 박스, 선명한 CTA를 사용합니다. 천원마켓도 등록 폼에서 같은 정도의 정돈감을 가져갈 수 있습니다.

## UX에서 디자인에 반영할 항목

UX 리서치에서 반영할 항목은 기능 추가가 아니라 시각 우선순위입니다. 검색, 필터, 채팅, 결제 같은 MVP 제외 기능은 넣지 않습니다.

### 1. 첫 화면은 현황보다 상품 스캔을 우선합니다.

홈 상단에 간단한 브랜드/입장 맥락은 주되, 화면의 주인공은 상품 사진 카드가 되어야 합니다. 상태 개수나 안내 문구를 넣더라도 카드 그리드를 밀어내는 큰 hero 형태는 피합니다.

근거: 천원마켓의 핵심 행동은 친구들이 올린 물건을 빠르게 훑고 예약 상태를 확인하는 것입니다. 현재 `src/app/page.tsx`도 헤더 아래 바로 `ProductList`를 렌더링합니다.

### 2. 상품 카드는 3초 안에 사진, 가격, 상태가 보여야 합니다.

카드에서 가장 먼저 보이는 정보 순서는 사진, 가격, 상태, 제목/메타 순으로 둡니다. 판매자와 예약자는 보조 정보로 낮추되, 예약중 상태는 배지로 즉시 보이게 합니다.

근거: `product-card.tsx`는 이미 `PRICE_DISPLAY_LABELS`와 `StatusBadge`를 함께 렌더링합니다. 디자인 개선은 이 정보의 위치와 무게를 조정하는 방향이면 충분합니다.

### 3. 상세 모달은 이미지와 예약 영역의 시각 분리를 유지합니다.

상세 모달에서는 대표 이미지/설명 영역과 예약자 선택 영역이 한 덩어리로 뭉치지 않게 합니다. 모바일에서는 예약 영역이 너무 아래로 밀리지 않도록 충분한 구분선, sticky 하단 영역, 또는 강한 섹션 헤더를 검토합니다.

근거: `product-detail-dialog.tsx`는 이미지 영역과 정보/예약 영역을 나란히 배치합니다. 모바일에서는 세로 흐름이 길어질 수 있으므로, 시각적 분리가 중요합니다.

### 4. 등록/수정 폼은 섹션 위계를 가져야 합니다.

폼은 한 화면의 긴 입력 묶음처럼 보이지 않게 사진, 기본 정보, 상태/메모, 비밀번호 순서의 섹션감만 부여합니다. 별도 wizard나 단계 전환은 만들지 않습니다.

근거: `product-form.tsx`는 현재 모든 입력을 한 form 안에 렌더링합니다. 구조는 유지하되 섹션 라벨, 여백, border, 업로드 영역 강조만으로 입력 부담을 낮출 수 있습니다.

## 적용 우선순위

1. 전역 색상 토큰 조정
2. 로고/파비콘 자산 정리
3. 버튼 primary 색상 변경
4. 상품 카드 색/배지/가격 위계 정리
5. 입장 화면에 로고와 민트 톤 적용
6. 등록/수정 폼의 업로드 영역과 에러/위험 액션 색 정리

근거: 색상 토큰과 버튼을 먼저 바꾸면 전체 인상이 빠르게 바뀝니다. 그 다음 카드와 입장 화면을 맞추면 사용자가 가장 먼저 보는 영역이 정리됩니다.

## 하지 않을 것

- 화면 전체를 진한 초록으로 덮지 않습니다.
- 보라/파랑 그라디언트나 SaaS 대시보드 톤을 쓰지 않습니다.
- 과한 그림자, 큰 hero, 마케팅 랜딩 페이지 스타일을 만들지 않습니다.
- 검색, 필터, 채팅, 결제 같은 MVP 제외 기능을 디자인 개선에 섞지 않습니다.
- 로고의 태그 안에 `1000`처럼 긴 숫자를 넣지 않습니다. 작은 크기에서 무너질 가능성이 큽니다.

## 완료 기준

- 첫 화면에서 민트 로고, 민트 CTA, 상품 사진 카드가 같은 브랜드로 보입니다.
- 16px 파비콘에서 흰색 `₩`가 식별됩니다.
- 상품 카드는 사진, 가격, 상태가 한눈에 구분됩니다.
- 판매중/예약중/거래완료 배지는 현재 의미를 유지합니다.
- 모바일 폭에서 버튼과 배지 텍스트가 겹치거나 잘리지 않습니다.
